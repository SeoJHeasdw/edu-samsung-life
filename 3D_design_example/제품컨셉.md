# RICE — 업무 방식을 기억하는 AI

## 왜 만들었나

지식노동자는 RFP 답변, 제안서 작성, 프레젠테이션 제작처럼 복잡하면서도 반복되는 업무에 AI를 쓴다.  
그런데 세션이 끝나는 순간, 대부분의 가치는 사라진다.

- 다음 작업에서 같은 배경과 절차를 **다시 설명**해야 한다
- 유용한 노하우는 개인 채팅과 메모 안에 **흩어진 채** 남는다
- 기존 AI는 사실은 기억해도, **업무를 수행하는 방법 자체**를 포착하거나 개선하지 않는다

RICE는 이 문제를 해결하기 위해 만들었다.

---

## 어떻게 작동하나

### Learning Loop — 사람이 통제하는 학습

작업이 끝나면 RICE는 다시 쓸 수 있는 노하우를 찾아 **Skill**로 제안한다.  
Skill에는 작업 단계, 확인 사항, 도구 사용법이 담긴다.

제안은 채팅 안의 **diff 카드**로 표시된다.  
사용자가 확인하고 승인해야만 Skill이 변경된다. 모르는 사이에 바뀌는 건 없다.

비슷한 요청이 들어오면 관련 Skill이 자동으로 선택되어 에이전트 컨텍스트에 추가된다.  
업무 방식이 바뀌면 동일한 승인 절차로 Skill 패치를 제안한다.

### 사용자 모델 — 매번 처음 만나는 챗봇이 아니라

RICE는 세션이 달라져도 유지되는 사용자 정보를 관리한다.

- 업무 환경, 선호 도구, 답변 형식 같은 선호를 기억
- 다음 대화에 자동으로 반영
- 결과적으로 업무 방식을 이해하는 동료에 가까워진다

### 조직 지식으로 확장

개인이 만든 학습 결과는 조직 전체로 퍼질 수 있다.

| 채널 | 내용 |
|---|---|
| **Skill Market** | 직원들이 재사용 가능한 업무 절차를 공유 |
| **Plugin Market** | 솔루션팀이 도메인 지식 + Skills + Connectors를 하나의 패키지로 배포 |

비개발자도 솔루션팀의 전문성을 즉시 활용할 수 있다.

---

## 학습이 눈에 보인다

RICE의 학습 증거는 보이지 않는 모델 가중치 변화가 아니다.

```
업무 수행 → 결과 검토 → 개선 제안 → 사용자 승인 → 다음 업무에서 재사용
```

이 흐름 전체가 사용자 눈앞에서 일어난다.

**데모 기준:** 사람이 25~30분 걸리는 작업(프레젠테이션 + 팀 초대 메일)을 약 1분 만에 완성.  
이후 슬라이드 구조 변경, 선호 템플릿, Executive Summary 추가 같은 피드백을 주면 — 그 피드백은 세션과 함께 사라지지 않고 다음 요청부터 반영된다.
