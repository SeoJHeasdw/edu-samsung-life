## 작업 원칙

- 이 지도를 먼저 읽고, 현재 과제와 관련된 파일만 연다.
- 사용자가 이미 바꾼 내용과 무관한 변경은 보존한다.
- 슬라이드 본문은 짧게 쓰고, 상세 설명은 발표자 노트에 둔다.
- 모델·제품·수치는 공식 출처나 1차 자료로 확인한다.
- 같은 문제가 반복되면 규칙이나 관련 문서 링크를 이 지도에 추가한다.

## 완료 기준

- 여는 태그와 닫는 태그 수가 맞는지 확인한다.
- `git diff --check`를 통과한다.
- 앱 실행과 화면 렌더링은 사용자가 직접 진행한다.
