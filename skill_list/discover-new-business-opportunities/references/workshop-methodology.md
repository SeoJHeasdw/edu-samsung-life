# 삼성생명 학습회용 신사업 발굴 실습

이 문서는 제공된 신사업 발굴 자료의 공개된 방법론만 교육용으로 재구성한다. block 처리된 내용과 실제 당사 평가 결과를 추정하거나 복원하지 않는다.

## 방법론을 Skill로 번역하기

| 업무 요소 | Skill 구성요소 | 이유 |
|---|---|---|
| 산업분류와 성장테마 | `references/` | 반복해서 참고할 분류체계와 조사 기준 |
| 섹터·BM 후보표 | `assets/*.csv` | 사람이 검토하고 수정할 입력 데이터 |
| 점수 합산과 순위 | `scripts/` | 모델의 암산 대신 재현 가능한 계산 |
| 조사·보고 순서 | `SKILL.md` | Codex가 따를 전체 업무 절차 |
| 단계별 승인 | Human Gate | 범위·기준·사업 판단을 사람이 책임 |

## 교육용 퍼널

실제 전체 규모를 그대로 재현하지 말고 작은 mock 퍼널로 구조를 체험한다.

```text
산업 후보 8개
→ 섹터 Scorecard
→ Short-list 5개
→ 밸류체인 × 고객문제 BM 후보
→ 시장성·적시성·적합성 채점
→ 1순위 BM과 추가 검증 항목
```

## 섹터 평가

100점 만점의 7개 기준을 사용한다.

| 기준 | 최고점 | 의미 |
|---|---:|---|
| 성장성 | 20 | 구조적 시장 성장 가능성 |
| 당사 경쟁력 | 20 | 기존 생명보험 역량·고객·채널 활용 가능성 |
| 수익성 | 15 | 영업이익과 반복매출 잠재력 |
| 전략연계 | 15 | 라이프케어·고객 생애가치와의 연결성 |
| 시장규모 | 10 | 의미 있는 사업 규모 가능성 |
| 진입용이성 | 10 | 규제, CAPEX, 인력, 시간 측면의 현실성 |
| 평판·재무 | 10 | 평판 리스크와 재무 부담의 안정성 |

이 점수는 사실 자체가 아니라 근거를 토대로 한 평가다. 기준별 근거가 없으면 점수를 확정하지 않는다.

## BM 발굴과 평가

섹터마다 `밸류체인 단계 × 고객문제` 매트릭스를 먼저 만들고, 기술명이 아니라 고객·지불자·수익모델이 있는 BM을 배치한다.

각 BM은 3점 만점의 세 축으로 평가한다.

- **시장성:** 시장이 충분히 크고 성장할 수 있는가?
- **적시성:** 상용 사례와 매출화 가시성이 있는가?
- **적합성:** 역량·규제·그룹 포트폴리오를 고려해 잘할 수 있는가?

교육용 분류:

- 8–9점: 1순위
- 6–7점: 2순위 또는 추가 검증
- 3–5점: 보류

점수 차이가 작으면 순위보다 근거 신뢰도와 다음 검증 행동을 우선한다.

## AI와 사람의 역할

### AI

- 자료 수집과 후보 확장
- 분류·표준화·중복 제거
- 점수 합산과 민감도 비교
- 반대 근거와 데이터 공백 표시
- 보고서 초안 작성

### 사람

- 조사 모집단과 제외 영역 확정
- 기준과 가중치 승인
- 내부 역량·규제·평판 판단
- 외부 전문가 검증 대상 선정
- Go/No-Go와 추진 방식 결정

## 3교시 운영

1. 제공 자료의 퍼널에서 반복 가능한 절차를 찾는다.
2. `분류체계`, `입력표`, `계산`, `보고 순서`, `Human Gate`로 분해한다.
3. 각각을 `references`, `assets`, `scripts`, `SKILL.md`에 매핑한다.
4. “에이전트 역할 이름”보다 역할이 따를 기준과 산출물 계약이 중요함을 확인한다.

## 4교시 실습

1. 번들 mock 데이터로 공통 Skill을 실행한다.
2. 상위 섹터와 1순위 BM, 근거 신뢰도를 확인한다.
3. 팀 업무에 맞지 않는 평가기준 하나를 찾아 수정한다.
4. Human Gate 하나를 추가하거나 강화한다.
5. 다시 실행해 변경 전후 순위와 판단 근거를 비교한다.
6. 실제 고객·회사·투자 데이터는 넣지 않는다.
